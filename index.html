Introduction

The Shodix API is a RESTful API that allows developers to integrate with the Shodix e-commerce platform. This API provides endpoints for managing users, stores, products, carts, orders, and messaging.

The API is built using Express.js and uses JWT for authentication. It also provides real-time messaging capabilities using Socket.io.

Base URL

http://localhost:3001

Getting Started

To get started with the Shodix API, you'll need to:

  1. Set up the server by installing the required dependencies
  2. Start the server on port 3001 (default)
  3. Register a user or store account
  4. Authenticate and obtain a JWT token
  5. Use the token to access protected endpoints

Installation

npm install express cors jsonwebtoken sequelize sqlite3 multer socket.io bcrypt

Starting the Server

node server.js

Authentication

The Shodix API uses JSON Web Tokens (JWT) for authentication. To access protected endpoints, you need to include the JWT token in the Authorization header.

Obtaining a Token

You can obtain a token by logging in as a user or store:

User Login

POST
/api/user/log
{
  "username": "your_username",
  "password": "your_password"
}

Store Login

POST
/api/store/log
{
  "store_name": "your_store_name",
  "password": "your_password"
}

Using the Token

Include the token in the Authorization header for all protected requests: 

// Example using fetch
fetch('/api/cart', {
  method: 'GET',
  headers: {
    'Authorization': 'Bearer YOUR_JWT_TOKEN',
    'Content-Type': 'application/json'
  }
})

User Endpoints

Register User

POST
/api/user/reg

Creates a new user account.

Request Body

{
  "username": "string",
  "password": "string",
  "full_location": "string"
}

Response

{
  "msg": "User created successfully! Please log in => Click here"
}

Login User

POST
/api/user/log

Authenticates a user and returns a JWT token.

Request Body

{
  "username": "string",
  "password": "string"
}

Response

{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}

Update User

PATCH
/api/user/update

Updates user information. Requires authentication.

Request Headers

Authorization: Bearer YOUR_JWT_TOKEN

Request Body

{
  "username": "string", // optional
  "password": "string", // optional
  "full_location": "string" // optional
}

Response

{
  "msg": "Updated successfully!"
}

Get Store

GET
/api/store/:store_name

Retrieves store information by store name.

Check Store Ownership

POST
/api/is/owner

Checks if the authenticated user is the owner of a specific store. Requires authentication.

Request Headers

Authorization: Bearer YOUR_JWT_TOKEN

Request Body

{
  "store_see": "string" // The name of the store to check ownership
}

Response

{
  "msg": "yes" // or "no" depending on ownership
}

WebSockets

The Shodix API uses Socket.io for real-time messaging between users and stores.

Chat Integration

To integrate with the real-time chat functionality, you need to:

  1. Connect to the Socket.io server
  2. Join a chat room
  3. Send and receive messages

Connecting to Socket.io

// Using the Socket.io client library
const socket = io('http://localhost:3001', {
  withCredentials: true
});

Joining a Chat Room

// Join a chat room
socket.emit('join_chat', chatId);

Sending Messages

// Send a message
socket.emit('send_message', {
  chatId: 1,
  message: 'Hello, world!',
  token: 'YOUR_JWT_TOKEN'
});

Receiving Messages

// Listen for incoming messages
socket.on('receive_message', (data) => {
  console.log('New message:', data);
  // data contains: msg_id, chat_id, message, timestamp
});
